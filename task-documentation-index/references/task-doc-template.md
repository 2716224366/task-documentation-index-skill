# Task Document Template

Use this shape when manually creating or repairing a task document.

```markdown
# <Task Title>

- Date: YYYY-MM-DD
- Kind: bug | feature | requirement | plan | asset | test | release | research | other
- Status: Planned | In progress | Done | Blocked | Superseded

## Purpose

Plain-language reason for the work.

## Index Check

- Index path read:
- Related prior docs checked:
- Reused decisions or lessons:

## Plan

- Intended approach.

## Work Log

- What actually happened.

## Files Or Systems Affected

- Paths, modules, documents, assets, APIs, or workflows.

## Verification

- Commands, browser checks, screenshots, manual QA, or reason verification was not possible.

## Outcome

- Final result.

## Follow-Ups Or Risks

- Remaining work, caveats, or decisions needed.
```
